Github Pagesで公開

  • tags:
  • lektor
  • quickstart
on 23-01-23

Lektorで生成される静的サイトは、多くのホスティングサービスが利用可能ですが、ここではGithub Pagesへのホスティング方法を説明します。

Githubへのユーザ登録は終了している前提です。Github Pagesでは２通りのURLのウェブページを作成できます。

  • https://ユーザー名.github.io
  • https://ユーザー名.github.io/リポジトリ名

1. https://ユーザー名.github.io

a) Githubの管理画面

ブラウザでGithubの管理画面へ行きます。Repositoriesの画面でレポジトリの作成(New)を選択し、「Repository name」に、ユーザー名.github.ioを入力します。「Add a README file」へチェックを入れ、下の方へスクロールして「Create repository」をクリックして作業を完了させます。

Lektorのデプロイ機能は、Masterブランチを対象にするので、defaultのブランチ名がMainの場合は、「branch」ボタンから鉛筆マークをクリックして、Masterにリネームします。

b) Lektor側の作業

ルートの設定ファイル**.lektorprojectへ下記の記述をします。

lektor-demo.lektorproject

[servers.ghpages]
target = ghpages://ユーザー名/ユーザー名.github.io

デプロイはLektorの管理画面の右上のアップロードマークをクリックするか、プロジェクトのディレクトリでlektor deployコマンドを実行します。

$ lektor deploy

2. ユーザ名.github.io/レポジトリ名

$ git branch -m main master
$ git fetch origin
$ git branch -u origin/master master
$ git remote set-head origin -a
